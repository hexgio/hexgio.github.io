一. 数据类型

void int char short float double...
其实在计算机里是没有数据类型的，说白了，就是访问内存的规则和这种规则的相关操作。
访问内存，你要遵循的方式。
注意: void 类型和无类型是两种概念，无类型只关注地址。
指针类型 指针并不能简单说是地址，它是一种类型！
int var = 300;
int *p = &var;  
数组  函数  结构体  --->都是一种数据类型  
int func_t (int x,int y);//声明
typedef int func_t (int, int);
func_t *p;

char * str_t;
typedef char *str_t;	
str_t a,b; ------char *a;  char *b;

struct node{
    int data;
    struct node *next;//可以的，结构体的大小确定
};
typedef struct node node_t;
node_t 是struct node的别名
typedef node_t *list_t;   ===== node * 
list_t head;

ARM core   Arm+安卓   mips   LA   powerpc   intel+win
A  R  M 
STM32F103

三 介绍开发板设备

AHB AHB1 AHB2 都是三总线，叫做高速总线
APB1 APB2 也是三总线，叫做外设总线

冯诺依曼体系结构: 取指令 取数据 用的是同一根总线
哈佛体系结构:  取指令 取数据 用的是不同的总线
cpu 是通过总线来访问外设的：
数据总线  地址总线  控制总线

想要给0x10001234 这个地址存储十进制的数据100

四 操作GPIO的输入输出功能

输入功能： 相当于检测（功能按键）
输出功能： 相当于控制 （LED 蜂鸣器）
写一个函数： 
1.功能
2.参数
3.返回值
在STM32中GPIO资源非常丰富
一共有5组GPIO,每组GPIO会有16管脚

通过芯片封装图可以了解64个管脚,每个管脚的定义
出现最多的就是GPIO管脚,其他的外设资源如何体现?
芯片的管脚是多功能复用型的管脚
一个管脚最多会有
    输入功能
        上拉输入		在默认情况下管脚呈现高电平
        下拉输入		在默认情况下管脚呈现低电平
        浮空输入		在默认情况下管脚状态不确定
    输出功能
        开漏输出		管脚只能输出低电平输出不了高电平
        推挽输出		管脚既可以输出低电平也可以输出高端平
    复用功能(timer USART IIC SPI)
        开漏复用		
        推挽复用
    模拟功能
        模拟输入功能	ADC(在模拟转换时会使用)
         
LED实验步骤：
1. 通过开发板原理图找到LED灯所连接的芯片的管脚
    LED0    	PC1
    LED1 	PC2
    LED2	PC3
    管脚输出高电平  灯亮
    管脚输出低电平  灯灭
2. 通过APB2总线使能GPIOC组的时钟
3. 初始化GPIO管脚
4. 控制管脚输出高低电平

悬赏任务:
1. 根据LED灯的驱动程序，完成流水灯 前4组  +4 
 0 1 2 
2. 根据LED灯的项目工程，拷贝一份，搞定蜂鸣器 +5
