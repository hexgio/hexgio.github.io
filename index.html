文件系统管理

使用mount命令挂载设备分区——使用方法：
mount [-t vfstype] [=o options] device dir
(1) -t vfstype #指定系统文件的类型。通常不必指定，mount会自动选择正确的类型。常用文件类型如下所示。
光盘或系统镜像：iso9660
DOS FAT 16文件系统：msdos
Windows 9x FAT 32 文件系统：vfat
windows NT NTFS 文件系统：ntfs
windows 文件共享：smbfs
Uinx(Linux)文件共享：nfs
(2) -o options #主要用来描述设备或文件的挂接方式。常用参数如下所示。
loop #用来把一个文件当成硬盘分区挂接上系统
ro #采用只读方式挂接设备
rw #采用读写方式挂接设备
iocharset #指定访问文件系统所采用字符集
(3) device #要挂载(mount)的设备
(4) dir #设备在系统上的挂载目录(mount point)

挂载光盘——示例：
要挂载光盘，首先需要使用mkdir命令来创建要挂载的目录，然后使用mount命令进行挂载，光盘的黑鹰设备名称是iso9660。如果要挂载的是光盘的镜像文件，需要添加-o loop参数

mkdir /mnt/cdrom1 #创建光盘的挂载目录
mount -t iso9660 /dev/cdrom /mnt/cdrom1 #挂载光盘到/mnt/cdrom1，/dev/cdrom是光盘驱动器的设备文件
ls /mnt/cdrom1 #sj查看光盘下的内容
mkdir /mnt/cdrom2 #创建光盘镜像的挂载目录
mount -0 loop -t iso9660 /pathtofile/you.iso /mnt/cdrom2 #挂载光盘镜像.iso文件，要加上” -o loop “进行挂载
ls /mnt/cdrom2 #查看光盘镜像下的内容
挂载U盘或者移动硬盘

注意：
插入U盘前后要分别用命令fdisk -l查看系统硬盘和硬盘分区情况。一般插入的硬盘的标识为sdb即为我们要挂载的硬盘或U盘， 创建挂载目录并进行挂载，挂载文件系统文件类型为vfat。如果要显示中文信息，增加参数-o iocharset=cp936`设定字符集是简体中文
例：挂载U盘或者移动硬盘：

mkdir -p /mnt/usb1 #创建挂载目录，-参数的作用是，如果目标目录的上级目录不存在，就一并创建
mount -t vfat /dev/sdb1 /mnt/usb1 #挂载U盘，文件系统是 vfat(W95 FAT32),/dev/sdb1是命令 fdisk -l 查到的设备名
ls /mnt/usb1 #查看光盘上的信息

ls /mnt/usb1 #查看已挂载设备上信息

ls /mnt/usb1 #再次查看挂载位置，应该是空目录了

    文件的包含   #include

    宏
    gcc -E hello.c -o hello.i

    gcc -S hello.i -o hello.s

    gcc -c hello.s -o hello.o

    链接： 将目标文件和库文件，运行时文件链接生成可执行文件

三 系统时钟
系统时钟并不是由STM32提供的，而是由ARM-CORTEX-M3 核心提供
系统定时器 （system timer | systick）
如果想要系统定时器的定时长短，需要从什么角度分析？
时钟频率 和 计数值

    1. 查找系统定时器的时钟频率
    打开《STM32F10x中文参考手册.pdf》
    P46 P47 图  6.2时钟  时钟树
    在STM32芯片中，一共有5钟时钟源
            HSI  系统内部的高速时钟
            HSE  系统外部的高速时钟
            PLL   锁相环（倍频器）
            LSI   系统内部的低速时钟
            LSE  系统外部的低速时钟
    系统定时器 的时钟频率是由SYSCLK提供
    SYSCLK的时钟频率可以由HSI HSE PLL
            但是HSI是8MHz  HSE也是8MHZ 都不足以达到72MHZ
            所以SYSCLK的时钟频率必须由PLL提供
            因为PLL是倍频器本身不能产生时钟频率 所以需要基础时钟源
    PLL的基础时钟源可以由  HSI HSE  提供
    因为外部的晶振要比芯片内部的时钟源稳定
    所以PLL的基础时钟源由  HSE 提供

            位带 ： 可以让程序员在操作外设寄存器的地址时，以原子操作的方式进行。
            是把位带区的每一位都做一个地址别名，映射到位带别名区
            将来程序员访问位带别名区，就相当于是访问位带区的某一位
            操作0x22000008的位带映射区，就可以直接访问你的位带的bit 2

            #define BITBAND (ADDR, BITNUM)
            ((ADDR & 0xf0000000)) + 0x20000000 + ((ADDR & 0xFFFFF) << 5) + BITNUM << 2 )

            char c = 13;  //00001101
            c = c << 2;  //00110100 ---> 52 = 13 * 4;
            按位左移 就是乘法 左移多少位就相当于乘以2的多少次方。
            //上述公式可以将位带区的地址转换为位带别名区的地址


            #define MEM_ADDR (ADDR)
            *(volatile unsigned int *) (ADDR)

一 操作GPIO的输入输出功能
芯片的管脚是多功能复用的
一个管脚最多会有四种功能
输出功能
开漏输出 管脚只能输出低电平输出不了高电平
推挽输出 管脚既可以输出低电平也可以输出高电平
输入功能
上拉输入 在默认的情况下呈现高电平
下拉输入 在默认的情况下呈现低电平
浮空输入 在默认的情况下管脚状态不确定
复用功能（timer USART IIC SPI…）
开漏复用
推挽复用
模拟功能
模拟输入功能 （ADC 在模拟转换时会使用）

Volatile 关键字的作用
告诉CPU，每次在取变量的值的时候，不要去缓存中取，每次都要到内存中重新加载变量的值。

二 GPIO的输入功能
KEY0 PC9
KEY1 PC8
KEY2 PA0
注意： 打开使能时钟时，需要将GPIOA组的时钟和C组的时钟都打开 |

当KEY0按下抬起后，LED0常亮，当KEY0再一次按下抬起的时候，LED0就灭了
当KEY1按下抬起后，LED1常亮，当KEY1再一次按下抬起的时候，LED1就灭了
当KEY2按下抬起后，LED2常亮，当KEY2再一次按下抬起的时候，LED2就灭了
