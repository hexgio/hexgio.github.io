三 介绍开发板设备

AHB AHB1 AHB2 都是三总线，叫做高速总线
APB1 APB2 也是三总线，叫做外设总线

冯诺依曼体系结构: 取指令 取数据 用的是同一根总线
哈佛体系结构:  取指令 取数据 用的是不同的总线
cpu 是通过总线来访问外设的：
数据总线  地址总线  控制总线

想要给0x10001234 这个地址存储十进制的数据100

四 操作GPIO的输入输出功能

输入功能： 相当于检测（功能按键）
输出功能： 相当于控制 （LED 蜂鸣器）
写一个函数： 
1.功能
2.参数
3.返回值
在STM32中GPIO资源非常丰富
一共有5组GPIO,每组GPIO会有16管脚

通过芯片封装图可以了解64个管脚,每个管脚的定义
出现最多的就是GPIO管脚,其他的外设资源如何体现?
芯片的管脚是多功能复用型的管脚
一个管脚最多会有
    输入功能
        上拉输入		在默认情况下管脚呈现高电平
        下拉输入		在默认情况下管脚呈现低电平
        浮空输入		在默认情况下管脚状态不确定
    输出功能
        开漏输出		管脚只能输出低电平输出不了高电平
        推挽输出		管脚既可以输出低电平也可以输出高端平
    复用功能(timer USART IIC SPI)
        开漏复用		
        推挽复用
    模拟功能
        模拟输入功能	ADC(在模拟转换时会使用)

expect

发表于 2022-06-17 更新于 2022-06-28

1.用expect实现与shell交互，用ssh连接termxu实例：

#!/usr/bin/expect #这一句为声明，声明的方式为# + ! + expect路径果不知道expectr的路径，可以执行which expect来查看
set host “192.168.0.0” #非本人ip，随便编的ip地址
set password “123456”
spawn ssh -p 8022 $host
expect
{

“yes/no\r” #如果检测到包含部分字符串，输出内容与shell交互
{
send “yes\r”
}

“*assword:”
{
send “$password\r”
}

}
interact #连接后保持连接状态

2.用expect登录termux并执行脚本实例：

#!/usr/bin/expect
set host “192.168.0.0”
set password “..Qq1ssh..”
set script “touch 54545455454545454544545545445455545454454454.sh” #假设脚本内容只是创建一个文件
spawn ssh -p 8022 $host
expect {
“yes/no\r”
{
send “yes\r”
}
“assword:”
{
send “$password\r”
}
}
expect ““ #匹配到任意字符
send “$script\r” #执行脚本
expect “*” #匹配到任意内容
send “exit\r” #退出
send “\r” #换行
expect eof #执行完后退出

3.expect eof与interact的区别：
这两个命令作为结束语句用的，如果只是执行一次任务就退出，那么可以用expect eof，如果连接后想要保持连接状态，那么就用interact

4.注意事项：
有一点需要注意，执行脚本的方式有很多种，但是执行expect脚本的时候只能用expect ./的方式执行，目前我知道的两种方式，不然会执行失败。
